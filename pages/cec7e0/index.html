npm 莫名其妙的问题及解决方法汇总

npm 在使用的时候，有时会遇到一些莫名其妙的问题，琢磨半天找不到头绪。最后又不知道怎么的就解决了。这里做一下记录，以便下次遇到查阅。

# 错误一：npm WARN tar ENOENT: no such file or directory

使用 npm install 安装依赖的时候，出现了一大堆这样的错误。

# 解决方法

具体原因不明，但是已经遇到过几次了，特别是用淘宝镜像。不知道是因为镜像源的问题还是自己网络与镜像源服务器连接的问题。

修改npm源为默认源，再次安装解决。

npm config set registry https://registry.npmjs.org/
1

# 错误二：ModuleBuildError: Module build failed

在 vue + element ui 的项目中遇到。更换了 node 版本之后，项目跑不起来。看错误信息大概能猜出是 sass 的问题。

Module build failed (from ./node_modules/mini-css-extract-plugin/dist/loader.js):
ModuleBuildError: Module build failed (from ./node_modules/sass-loader/dist/cjs.js):
Error: Missing binding /Users/xxx/ xxx/node_modules/node-sass/vendor/darwin-x64-64/binding.node Node Sass could not find a binding for your current environment: OS X 64-bit with Node.js 10.x
Found bindings for the following environments: - OS X 64-bit with Node.js 12.x
This usually happens because your environment has changed since running `npm install`. Run `npm rebuild node-sass` to download the binding for your current environment.

# 解决方法

其实错误提示中有提到，重新构建 node-sass 解决。
